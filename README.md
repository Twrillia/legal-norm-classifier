# 한국 법규범의 규칙·원칙 자동 분류 시스템

한국 현행 법령 5,557개(511,145개 항)를 대상으로 각 조항을 **규칙(rule)**, **원칙(principle)**, **해당없음**의 세 범주로 자동 분류하고, 정규식 기반 접근법과 LLM 기반 접근법의 성능을 비교·분석한 연구입니다.

## 연구 배경

김도균 교수의 법적 이익형량 이론(서울대학교 법학 제48권 제2호)에 기반하여, 법규범을 확정적 법규범(규칙)과 법원리(원칙)로 구분하는 이원적 규범 유형론을 NLP 기법으로 조작화(operationalize)하였습니다.

| 구분 | 확정적 법규범 (규칙) | 법원리 (원칙) |
|------|---------------------|--------------|
| 구조 | 명확한 구성요건 → 확정적 법률효과 | 추상적 가치 → 최적화 명령 |
| 적용 방식 | 전부 아니면 무 (all-or-nothing) | 정도의 차이 (0~100% 실현) |
| 법적 추론 | 포섭 추론 (subsumption) | 이익형량 (balancing) |

## 분류 결과

### 전체 법령 분류 (정규식 기반, N=511,145)

| 분류 | 건수 | 비율 |
|------|------|------|
| 규칙 | 390,534건 | 76.4% |
| 해당없음 | 112,863건 | 22.1% |
| 원칙 | 7,748건 | 1.5% |

### 정규식 vs LLM 비교 (N=300)

- **일치율**: 77.0% (231/300)
- LLM(Gemini 2.5 Flash-Lite)은 정규식 대비 원칙을 2.5배 더 많이 식별
- 불일치는 주로 형식과 내용이 괴리되는 경계 영역 조항에 집중

## 기술 스택

- **언어**: Python
- **데이터 수집**: 국가법령정보센터 Open API (law.go.kr)
- **정규식 분류**: Python 정규식 기반 55개 패턴 (해당없음 15개, 원칙 21개, 규칙 19개)
- **LLM 분류**: Google Gemini 2.5 Flash-Lite (temperature 0.1)
- **데이터 규모**: 5,557개 법령, 511,145개 항
